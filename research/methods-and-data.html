Evidence and reproducibility

Methods and data

The public repository separates source records, analytical groupings, generated results, and checks so changes can be reviewed and reproduced.

Canonical records

The clean record table is the stable source for the analysis. Each row carries a persistent identifier and documented fields, allowing a correction or addition to be discussed precisely.

Exposure groups

Exposure groups make analytical assumptions explicit instead of hiding them inside calculations. The O5 and O6 distribution route remains separate from the transaction-network route because it answers a different borrower-side question.

Reproducible calculations

Scripts rebuild the published outputs from the canonical inputs. Validation checks catch broken identifiers, invalid categories, missing required values, duplicate records, and inconsistent references before results are accepted.

Contributing from a fork

A contributor can fork the repository and propose either new records or corrections. Automated checks run on the proposed branch even when the contributor does not directly edit the canonical dataset. Maintainers can review the evidence, normalize the proposed change, and merge it without requiring write access.
